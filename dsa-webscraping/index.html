Web Scraping is the automated process of extracting information from websites. Instead of manually copying data, a scraper reads and parses the webpage's code to collect the needed data.

‚ùå When NOT to Use Web Scraping

üìù HTML Basics for Web Scraping

What is HTML?

HTML (HyperText Markup Language) structures content on the web. It's made up of elements (tags) like <div>, <p>, <h1>, <a>, etc.

Basic HTML Structure

<!DOCTYPE html> <html> <head> <title>Sample Page</title> </head> <body> <h1>Welcome!</h1> <p>This is a sample page.</p> </body> </html>

Key HTML Elements

<!DOCTYPE html>: Declares the document type

<html>: Root element of the page

<head>: Contains metadata, styles, and scripts

<body>: Contains visible content

Important HTML Tags for Scraping

Tag Meaning Common Use
<div> Division/Container Group content
<p> Paragraph Text blocks
<h1>-<h6> Headings Titles and sections
<a> Anchor (links) URLs, navigation
<img> Image Pictures and icons
<table> Table Structured tabular data
<ul>, <ol>, <li> Lists List items
<span> Inline container Inline styling

üéØ Attributes Matter!

HTML tags often have attributes like id, class, href, src.

We use these attributes to target the correct elements.

<p class="price">$29.99</p>

Here, the class="price" attribute helps identify the price on the page.

Common HTML Attributes

Attribute Purpose
href Hyperlink reference
src Image or media source
class CSS class (commonly used for scraping)
id Unique identifier for an element
name Often used in form elements
type Used in <input> tags

Understanding Nested Elements

HTML is hierarchical. Tags can contain other tags.

<div class="article"> <h2>Title</h2> <p>This is a summary.</p> </div>

To extract both the title and summary, first locate the parent <div class="article">, then its children.

üí° Practical Tips

  • Use browser DevTools (Right-click ‚Üí Inspect) to examine HTML structure
  • Target elements with unique id or descriptive class attributes
  • Use tag nesting logic to extract specific parts of a page

üé® CSS Selectors for Scraping

What is CSS?

CSS (Cascading Style Sheets) controls the style and layout of HTML elements. For scraping, we mainly care about CSS selectors to find and extract data.

Common CSS Selectors

Selector Meaning Example
.class Selects elements by class .price, .title
#id Selects an element by id #main, #product-title
tag Selects all elements of a tag h1, div, p
tag.class Selects a tag with class p.price, div.container

Example: Selecting Elements

HTML:

<p class="price">$29.99</p>

CSS selector to target this:

p.price

In Python using BeautifulSoup:

soup.select_one("p.price")

Navigating HTML in Scraping

Scraping tools like BeautifulSoup use tag names and attributes to locate elements.

Example Targets:

  • By tag: soup.find('div')
  • By class: soup.find('div', class_='product')
  • By ID: soup.find(id='header')
  • By attribute: soup.find('a', {'href': True})

üåê Using Requests for Web Scraping

What is requests?

requests is a Python library used to send HTTP requests easily. It allows you to fetch the content of a webpage programmatically. It is commonly used as the first step before parsing HTML with BeautifulSoup.

üì¶ Installing requests

pip install requests

üöÄ Sending a Basic GET Request

import requests url = "https://example.com" response = requests.get(url) # Print the HTML content print(response.text)

url: The website you want to fetch

response.text: The HTML content of the page as a string

‚úÖ Checking the Response Status

print(response.status_code) if response.status_code == 200: print("Page fetched successfully!") else: print("Failed to fetch the page.")

Common Status Codes

Code Meaning
200 OK (Success)
404 Not Found
403 Forbidden
500 Internal Server Error

Important Response Properties

Property Description
response.text HTML content as Unicode text
response.content Raw bytes of the response
response.status_code HTTP status code
response.headers Metadata like content-type, server info

üé≠ Adding Headers to Mimic a Browser

Sometimes websites block automated requests. Adding a User-Agent header helps the request look like it is coming from a real browser.

headers = { "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)" } response = requests.get(url, headers=headers)

‚ö†Ô∏è Handling Connection Errors

try: response = requests.get(url, timeout=5) response.raise_for_status() # Raises HTTPError for bad responses print(response.text) except requests.exceptions.RequestException as e: print(f"An error occurred: {e}")

‚ú® Best Practices for Fetching Pages

  • Always check the HTTP status code
  • Use proper headers to mimic a browser
  • Set a timeout to avoid hanging indefinitely
  • Respect the website by not making too many rapid requests

üìö Summary

  • requests makes it simple to fetch web pages using Python
  • It is the starting point for most web scraping workflows
  • Combining requests with BeautifulSoup allows for powerful data extraction

üçú Using BeautifulSoup for Web Scraping

What is BeautifulSoup?

BeautifulSoup is a Python library used to parse HTML and XML documents. It creates a parse tree from page content, making it easy to extract data. It is often used with requests to scrape websites.

üì¶ Installing BeautifulSoup

pip install beautifulsoup4 lxml

üé® Creating a BeautifulSoup Object

from bs4 import BeautifulSoup import requests url = "https://example.com" response = requests.get(url) soup = BeautifulSoup(response.text, "lxml")

response.text: HTML content

"lxml": A fast and powerful parser (you can also use "html.parser")

üå≥ Understanding the HTML Structure

BeautifulSoup treats the page like a tree. You can search and navigate through tags, classes, ids, and attributes.

<html> <body> <h1>Title</h1> <p class="description">This is a paragraph.</p> <a href="/page">Read more</a> </body> </html>

üîç Common Methods in BeautifulSoup

1. Accessing Elements

# Access the first occurrence of a tag soup.h1 # Get the text inside a tag soup.h1.text

2. find() Method

Finds the first matching element:

soup.find("p") # Find a tag with specific attributes soup.find("p", class_="description")

3. find_all() Method

Finds all matching elements:

soup.find_all("a")

4. Using select() and select_one()

Select elements using CSS selectors:

soup.select_one("p.description") soup.select("a")

üîó Extracting Attributes

Get the value of an attribute, such as href from an <a> tag:

link = soup.find("a") print(link["href"]) # Or using .get() print(link.get("href"))

üå≤ Traversing the Tree

Method Purpose Example
.parent Access parent element soup.p.parent
.children Access child elements list(soup.body.children)
.find_next_sibling() Find next sibling soup.h1.find_next_sibling()

‚úÖ Handling Missing Elements Safely

Always check if an element exists before accessing it:

title_tag = soup.find("h1") if title_tag: print(title_tag.text) else: print("Title not found")

üéØ Summary

  • BeautifulSoup helps parse and navigate HTML easily
  • Use .find(), .find_all(), .select(), and .select_one() to locate data
  • Always inspect the website's structure before writing scraping logic
  • Combine BeautifulSoup with requests for full scraping workflows
