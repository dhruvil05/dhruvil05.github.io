JavaScript Fundamentals (ES6+ Features)

let/const, arrow functions, template literals â–¼

Modern variable declarations where let allows reassignment and const creates immutable bindings.

Arrow functions () => {} provide concise syntax and lexical this binding.

Template literals use backticks for string interpolation: `Hello ${name}`

Destructuring, Rest/Spread Operators â–¼

Destructuring extracts values from objects and arrays efficiently.

const {name, age} = user
const [first, second] = array

Spread operator expands arrays/objects: [...arr1, ...arr2]

Rest operator collects remaining elements: function sum(...numbers) {}

Promises and Async/Await â–¼

Promises handle asynchronous operations with chainable methods.

.then()
.catch()
.finally()
Promise.all()
Promise.race()
Promise.resolve()
Promise.reject()

Async/await provides synchronous-looking async code: await fetch(url)

Modules: CommonJS (require) vs ES Modules (import) â–¼

CommonJS:

const module = require('module')
module.exports = {}

ES Modules:

import { func } from 'module'
export default
export const

ES modules are static and support tree-shaking for optimization.

Core Node.js Concepts

Event Loop & Async Programming â–¼

Node.js runs on a single-threaded event loop that handles async operations efficiently.

Phases: timers, pending callbacks, poll, check, close callbacks

process.nextTick()
setImmediate()
setTimeout()
Callbacks, Promises, and Async/Await â–¼

Callbacks: Traditional pattern for async operations

fs.readFile(path, (err, data) => {})

Promises: Modern async handling with chainable .then()

Async/await: Syntactic sugar over promises for cleaner, more readable code

Streams and Buffers â–¼

Streams process data in chunks (Readable, Writable, Duplex, Transform).

.pipe()
.on('data')
.on('end')
.read()
.write()

Buffers handle binary data efficiently:

Buffer.from()
Buffer.alloc()
.toString()

fs.readFile()
fs.readFileSync()
fs.createReadStream()

Write files:

fs.writeFile()
fs.writeFileSync()
fs.createWriteStream()

Directory operations:

fs.readdir()

File info:

fs.stat()

Access process info:

Control execution:

Environment Variables â–¼

Access via process.env.VARIABLE_NAME

Set using .env files with packages like dotenv

Used for configuration such as database URLs, API keys, and port numbers.
