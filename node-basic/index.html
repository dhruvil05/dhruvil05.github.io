04 // ASYNC_PATTERNS

CONTROL_FLOW
ERROR_FIRST

Callback Pattern

// The Pyramid of Doom
getData(id, (err, data) => {
if (err) return handle(err);
process(data, (err, result) => {
if (err) return handle(err);
save(result, (err) => {
if (err) return handle(err);
console.log('Done');
});
});
});

The original Node.js pattern. Every callback receives error as first argument.

‚ö†Ô∏è Callback Hell
Deeply nested callbacks reduce readability and error handling becomes fragmented.
CHAINABLE

Promise Chain

fetchUser(id)
.then(user => fetchOrders(user.id))
.then(orders => processOrders(orders))
.then(result => {
console.log(result);
return result;
})
.catch(err => handleError(err))
.finally(() => cleanup());
// Parallel execution
const [users, posts] = await Promise.all([
fetchUsers(),
fetchPosts()
]);
Promise.all()
Wait for all promises. Fails fast on first rejection.
Promise.race()
Returns first settled promise (fulfilled or rejected).
Promise.allSettled()
Wait for all, never rejects. Returns status array.
SYNCHRONOUS_SYNTAX

Async / Await

async function getUserData(userId) {
try {
const user = await fetchUser(userId);
const orders = await fetchOrders(user.id);
const processed = await process(orders);
return processed;
} catch (err) {
console.error('Failed:', err);
throw new Error('User fetch failed');
}
}
// Top-level await (ES2022)
const data = await fetchData();
‚úì Error Handling
Standard try/catch blocks instead of .catch() chains. Stack traces are cleaner.
‚úì Debugging
Set breakpoints inside async functions. Step through like synchronous code.
‚úì Conditionals
Easy to use if statements and loops with async operations.
