JavaScript Fundamentals

Modern ES6+ features essential for Node.js development

Variable Declarations

ES6

Use let for reassignable values and const for immutable bindings. Avoid var.

variables.js 
// Mutable variable
let count = 0;
count = 1; // ✓ Valid

// Immutable binding
const API_URL = 'https://api.example.com';
// API_URL = 'new'; // ✗ TypeError

// const objects can be modified
const user = { name: 'John' };
user.name = 'Jane'; // ✓ Valid

Arrow Functions

Syntax

Concise syntax with lexical this binding. Perfect for callbacks and array methods. No own this, arguments, or super.

arrows.js 
// Traditional function
const add = function(a, b) {
  return a + b;
};

// Arrow function
const add = (a, b) => a + b;

// With implicit return
const numbers = [1, 2, 3];
const doubled = numbers.map(n => n * 2);

Template Literals

Strings

Backtick strings enabling interpolation and multiline text. Supports embedded expressions with ${} syntax.

templates.js 
const name = 'Node.js';
const version = 18;

// String interpolation
const message = `Welcome to ${name} v${version}`;

// Multiline strings
const html = `
  <div class="container">
    <h1>${name}</h1>
  </div>
`;

// Tagged templates
const sql = SQL`SELECT * FROM users WHERE id = ${userId}`;

Destructuring

Patterns

Extract values from objects and arrays into distinct variables. Supports nesting, renaming, and default values for cleaner code.

destructure.js 
// Object destructuring
const { name, age: years, role = 'user' } = user;

// Array destructuring
const [first, second, ...rest] = array;

// Nested destructuring
const { user: { email } } = data;

// Function parameters
function Component({ title, content }) {
  return `<h1>${title}</h1>`;
}

Spread & Rest Operators

Operators

Spread (...): Expands iterables into individual elements. Use for copying arrays/objects or passing arguments.

spread.js 
// Array concatenation
const combined = [...arr1, ...arr2];

// Object merging
const defaults = { ...config, port: 3000 };

// Function arguments
const nums = [1, 2, 3];
Math.max(...nums); // 3

Rest (...): Collects remaining elements into an array. Perfect for variadic functions.

rest.js 
// Function parameters
function sum(...numbers) {
  return numbers.reduce((a, b) => a + b, 0);
}

// Destructuring
const [head, ...tail] = [1, 2, 3, 4];
// head = 1, tail = [2,3,4]

Asynchronous Patterns

Handling async operations from callbacks to async/await

Promises

Objects representing eventual completion or failure of asynchronous operations. Chainable methods for handling success (.then()) and errors (.catch()).

promises.js 
const fetchData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      success ? resolve(data) : reject(error);
    }, 1000);
  });
};

// Chaining
fetchData()
  .then(data => processData(data))
  .then(result => console.log(result))
  .catch(err => console.error(err))
  .finally(() => console.log('Done'));

// Promise combinators
const results = await Promise.all([promise1(), promise2()]);
const winner = await Promise.race([p1(), p2()]);

Async/Await

Recommended

Syntactic sugar over Promises for writing asynchronous code that looks synchronous. Makes error handling with try/catch more intuitive.

async-await.js 
async function getUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);
    
    if (!response.ok) {
      throw new Error('User not found');
    }
    
    const data = await response.json();
    return data;
    
  } catch (error) {
    console.error('Fetch failed:', error);
    throw error;
  }
}

Core Node.js Concepts

Event loop, streams, and buffers explained

The Event Loop

Node.js runs on a single-threaded event loop that enables non-blocking I/O operations. Understanding the phases is crucial for performance optimization.

Timers

setTimeout, setInterval

Pending Callbacks

I/O callbacks deferred to next loop

Poll

Retrieve new I/O events; execute callbacks

Check

setImmediate callbacks

event-loop.js 
// Execution order demonstration
console.log('1. Start');

setTimeout(() => console.log('2. Timeout'), 0);
setImmediate(() => console.log('3. Immediate'));
process.nextTick(() => console.log('4. Next Tick'));

console.log('5. End');

// Output: 1 → 5 → 4 → 2 → 3

process.nextTick()

Executes before next event loop phase. Highest priority.

process.nextTick(callback)

setImmediate()

Executes after I/O events in check phase.

setImmediate(callback)

Best Practice

Use setImmediate over process.nextTick for I/O-bound operations to prevent starvation.

Streams

Abstract interfaces for handling streaming data in chunks. Memory-efficient for large files or network operations. Types: Readable, Writable, Duplex, Transform.

streams.js 
const fs = require('fs');

// Reading with streams
const readStream = fs.createReadStream('large-file.txt', { encoding: 'utf8' });

readStream.on('data', (chunk) => {
  console.log('Received chunk:', chunk.length);
});

readStream.on('end', () => console.log('Finished reading'));

// Piping streams
fs.createReadStream('input.txt')
  .pipe(fs.createWriteStream('output.txt'));

Buffers

Fixed-size chunks of memory for handling binary data directly. Essential for file operations and network protocols.

Buffer.from(string)

Create buffer from string

Buffer.alloc(size)

Allocate zero-filled buffer

buffer.toString()

Convert to string

Module Systems

CommonJS vs ES Modules

CommonJS

Legacy

Synchronous module system used in Node.js since inception. Loaded at runtime with dynamic require().

commonjs/ 
// Importing
const fs = require('fs');
const { helper } = require('./utils');

// Exporting
module.exports = myFunction;
// or
exports.helper = helper;

Note: Still default in Node.js. Use .cjs extension or configure in package.json.

ES Modules

Modern

Standard JavaScript module system with static analysis and tree-shaking support. Use "type": "module" in package.json.

esm/ 
// Importing
import fs from 'fs';
import { helper } from './utils.js';
import * as utils from './utils.js';

// Exporting
export const helper = () => {};
export default mainFunction;

Advantages: Static analysis, tree-shaking, top-level await, better browser compatibility.

File System (fs)

Reading, writing, and managing files

Read Files

  • fs.readFile() Async
  • fs.readFileSync() Sync
  • fs.createReadStream() Stream
const data = await fs.readFile('file.txt', 
  'utf8'
);

Write Files

  • fs.writeFile() Async
  • fs.writeFileSync() Sync
  • fs.appendFile() Append
await fs.writeFile('file.txt', 
  'Hello World'
);

Directories

  • fs.readdir() List
  • fs.mkdir() Create
  • fs.rmdir() Remove
const files = await fs.readdir('./');

File Information

  • fs.stat() Metadata
  • fs.access() Permissions
  • fs.unlink() Delete

fs/promises API (Recommended)

Async/Await

Modern promise-based API eliminating callback hell. Import from fs/promises for cleaner async code.

fs-promises.js 
import { readFile, writeFile, mkdir } from 'fs/promises';

async function setupProject() {
  try {
    await mkdir('./logs', { recursive: true });
    await writeFile('./logs/app.log', 'Session started');
    
    const data = await readFile('./config.json', 'utf8');
    return JSON.parse(data);
    
  } catch (err) {
    console.error('Setup failed:', err);
  }
}

Environment Variables

Access via process.env.VARIABLE_NAME. Use dotenv package for local development.

.env

config.js 
import 'dotenv/config';

const config = {
  port: process.env.PORT || 3000,
  env: process.env.NODE_ENV || 'development',
  dbUrl: process.env.DB_URL,
  isDev: process.env.NODE_ENV === 'development'
};

export default config;

Process Control

process.exit()

Terminate process with code

process.kill()

Send signal to process

process-control.js 
// Graceful shutdown
process.on('SIGTERM', () => {
  console.log('SIGTERM received, closing...');
  server.close(() => {
    process.exit(0);
  });
});

// Uncaught exceptions
process.on('uncaughtException', (err) => {
  console.error('Uncaught:', err);
  process.exit(1);
});

Child Processes

Spawn subprocesses for CPU-intensive tasks or shell commands.

exec() Buffer output, shell commands
spawn() Streaming, large data
fork() New Node.js instance
child-process.js 
import { exec, spawn } from 'child_process';

// Execute shell command
exec('ls -la', (err, stdout) => {
  console.log(stdout);
});

// Spawn with streams
const child = spawn('node', ['script.js']);
child.stdout.on('data', (data) => console.log(data));
