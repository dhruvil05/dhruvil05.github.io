Complete Guide to File and Directory Operations

Introduction to Node.js File System

The file system (fs) module in Node.js provides an API for interacting with the file system in a way modeled on standard POSIX functions. It allows you to read, write, update, delete, and manipulate files and directories on your computer.

Getting Started

To use the file system module, you need to require it:

const fs = require('fs'); // Callback-based const fsPromises = require('fs').promises; // Promise-based const fs = require('fs/promises'); // Promise-based (Node.js 14+)

Three Ways to Use fs Module:
  • Synchronous: Blocks execution until operation completes
  • Callbacks: Non-blocking with callback functions
  • Promises: Non-blocking with async/await syntax

Reading Files

Synchronous Reading (Blocking)

Blocks execution until the file is read completely.

const fs = require('fs'); try { const data = fs.readFileSync('example.txt', 'utf8'); console.log(data); } catch (error) { console.error('Error reading file:', error); }

Asynchronous with Callbacks

Non-blocking operation with callback function.

fs.readFile('example.txt', 'utf8', (error, data) => { if (error) { console.error('Error reading file:', error); return; } console.log(data); });

Asynchronous with Promises (Recommended)

Clean async/await syntax for better readability.

const fs = require('fs/promises'); async function readFileExample() { try { const data = await fs.readFile('example.txt', 'utf8'); console.log(data); } catch (error) { console.error('Error reading file:', error); } }

Other Reading Operations

Check File Existence

async function checkExists() { try { await fs.access('example.txt'); console.log('File exists'); } catch { console.log('File does not exist'); } }

Get File Information

const stats = await fs.stat('example.txt'); console.log(stats.isFile()); // true if it's a file console.log(stats.isDirectory()); // true if it's a directory console.log(stats.size); // file size in bytes console.log(stats.mtime); // last modified time console.log(stats.birthtime); // creation time
Reading JSON Files
async function readJSON(filename) { const data = await fs.readFile(filename, 'utf8'); return JSON.parse(data); } const config = await readJSON('config.json');

Writing Files

Write File (Overwrites existing content)

const fs = require('fs/promises'); async function writeFileExample() { try { await fs.writeFile('output.txt', 'Hello World!', 'utf8'); console.log('File written successfully'); } catch (error) { console.error('Error writing file:', error); } }

Append to File (Adds to existing content)

await fs.appendFile('log.txt', 'New log entry\n', 'utf8');

Copy File

await fs.copyFile('source.txt', 'destination.txt'); console.log('File copied');

Rename/Move File

await fs.rename('old-name.txt', 'new-name.txt'); // Can also move files to different directories await fs.rename('file.txt', './subfolder/file.txt');

Delete File

await fs.unlink('file-to-delete.txt'); console.log('File deleted');
Writing JSON Files
async function writeJSON(filename, data) { const jsonString = JSON.stringify(data, null, 2); await fs.writeFile(filename, jsonString, 'utf8'); } await writeJSON('data.json', { name: 'John', age: 30 });
Simple Logger Example
async function log(message) { const timestamp = new Date().toISOString(); const logEntry = `[${timestamp}] ${message}\n`; await fs.appendFile('app.log', logEntry, 'utf8'); } await log('Application started');

Directory Operations

Create Directory

// Create single directory await fs.mkdir('new-folder'); // Create nested directories await fs.mkdir('path/to/nested/folder', { recursive: true });

Read Directory Contents

// Get array of filenames const files = await fs.readdir('./my-folder'); console.log(files); // ['file1.txt', 'file2.txt', 'subfolder'] // Get detailed information const filesWithTypes = await fs.readdir('./my-folder', { withFileTypes: true }); filesWithTypes.forEach(file => { console.log(file.name, file.isDirectory() ? 'DIR' : 'FILE'); });

Remove Directory

// Remove empty directory await fs.rmdir('empty-folder'); // Remove directory and all contents (Node.js 14.14+) await fs.rm('folder-with-files', { recursive: true, force: true });
Recursive Directory Listing
async function listFilesRecursively(dir, fileList = []) { const files = await fs.readdir(dir, { withFileTypes: true }); for (const file of files) { const filePath = path.join(dir, file.name); if (file.isDirectory()) { await listFilesRecursively(filePath, fileList); } else { fileList.push(filePath); } } return fileList; }

Working with Paths

Often used with the path module:

const path = require('path'); // Join paths correctly for any OS const filePath = path.join(__dirname, 'data', 'file.txt'); // Get file extension const ext = path.extname('document.pdf'); // '.pdf' // Get filename without extension const name = path.basename('document.pdf', '.pdf'); // 'document' // Get directory name const dir = path.dirname('/path/to/file.txt'); // '/path/to' // Resolve absolute path const absolute = path.resolve('file.txt');

Working with Streams

Streams are useful for handling large files efficiently without loading the entire file into memory.

Reading Streams

const fs = require('fs'); const readStream = fs.createReadStream('large-file.txt', 'utf8'); readStream.on('data', (chunk) => { console.log('Received chunk:', chunk); }); readStream.on('end', () => { console.log('Finished reading file'); }); readStream.on('error', (error) => { console.error('Error:', error); });

Writing Streams

const writeStream = fs.createWriteStream('output.txt'); writeStream.write('First line\n'); writeStream.write('Second line\n'); writeStream.end('Final line\n'); writeStream.on('finish', () => { console.log('Finished writing'); });

Piping Streams (Copy files efficiently)

const readStream = fs.createReadStream('source.txt'); const writeStream = fs.createWriteStream('destination.txt'); readStream.pipe(writeStream); writeStream.on('finish', () => { console.log('File copied successfully'); });
Why Use Streams?
  • Handle large files without loading them entirely into memory
  • Process data as it arrives, chunk by chunk
  • More efficient for reading/writing large amounts of data
  • Can pipe data from one stream to another

File Watching

Monitor files and directories for changes:

Watch a File

const fs = require('fs'); // Watch a single file fs.watch('example.txt', (eventType, filename) => { console.log(`Event: ${eventType} on ${filename}`); }); // Watch a directory fs.watch('./my-folder', { recursive: true }, (eventType, filename) => { console.log(`${filename} was ${eventType}`); });

Watch File with Details

fs.watchFile('example.txt', (current, previous) => { console.log(`File modified at: ${current.mtime}`); console.log(`Previous modified: ${previous.mtime}`); });

Advanced File Operations

Reading Files in Chunks

const fs = require('fs/promises'); async function readInChunks(filename, chunkSize = 1024) { const fileHandle = await fs.open(filename, 'r'); const buffer = Buffer.alloc(chunkSize); try { let bytesRead; do { const result = await fileHandle.read(buffer, 0, chunkSize); bytesRead = result.bytesRead; if (bytesRead > 0) { console.log(buffer.toString('utf8', 0, bytesRead)); } } while (bytesRead > 0); } finally { await fileHandle.close(); } }

Working with File Descriptors

const fileHandle = await fs.open('example.txt', 'r'); try { const stats = await fileHandle.stat(); console.log('File size:', stats.size); const buffer = Buffer.alloc(100); await fileHandle.read(buffer, 0, 100, 0); console.log(buffer.toString()); } finally { await fileHandle.close(); }

Truncating Files

// Truncate to specific size await fs.truncate('file.txt', 100); // Keep only first 100 bytes // Empty a file await fs.truncate('file.txt', 0);

Changing File Permissions

// Make file readable and writable by owner only await fs.chmod('file.txt', 0o600); // Make file executable await fs.chmod('script.sh', 0o755);
Permission Codes Explained:
  • 0o600: Owner can read and write
  • 0o644: Owner can read/write, others can read
  • 0o755: Owner can read/write/execute, others can read/execute
  • 0o777: Everyone can read/write/execute (not recommended)

Error Handling

Proper Error Handling Pattern

const fs = require('fs/promises'); async function safeFileOperation() { try { const data = await fs.readFile('file.txt', 'utf8'); return data; } catch (error) { if (error.code === 'ENOENT') { console.error('File not found'); } else if (error.code === 'EACCES') { console.error('Permission denied'); } else if (error.code === 'EISDIR') { console.error('Expected file but found directory'); } else { console.error('Unexpected error:', error); } throw error; } }

Common Error Codes

ENOENT

File or directory doesn't exist

EACCES

Permission denied

EISDIR

Expected a file but got a directory

ENOTDIR

Expected a directory but got a file

EEXIST

File or directory already exists

EMFILE

Too many open files

Important Error Handling Tips:
  • Always use try-catch blocks with async operations
  • Check error codes to provide specific error messages
  • Close file handles in finally blocks to prevent leaks
  • Validate file paths before performing operations
  • Handle both synchronous and asynchronous errors appropriately

Sync vs Async - When to Use What

Scenario Recommended Approach Reason
Application startup/initialization Synchronous (Sync) App needs config before starting
Reading config files at startup Synchronous (Sync) Sequential execution required
One-time scripts Synchronous (Sync) Scripts run once and exit
Web servers and APIs Asynchronous (Async) Handle concurrent requests
Multiple operations Asynchronous (Async) Don't block event loop
Long-running applications Asynchronous (Async) Performance and scalability

Performance Optimization Tips

1. Buffer Size Matters

Choose appropriate buffer sizes for streams. Too small = many operations, too large = memory waste.

// Good for most cases const stream = fs.createReadStream('file.txt', { highWaterMark: 64 * 1024 // 64KB chunks });
2. Cache File Stats

If you need to check file stats multiple times, cache the results.

const stats = await fs.stat('file.txt'); // Use stats multiple times without re-reading if (stats.isFile() && stats.size > 1000000) { // Process large file }
3. Use Appropriate Methods

Choose the right method for your use case:

  • Small files: readFile/writeFile
  • Large files: createReadStream/createWriteStream
  • Specific portions: open() with read()
Common Mistakes to Avoid:
  • Using sync methods in web servers/APIs
  • Not closing file handles after opening them
  • Reading entire large files into memory
  • Not handling errors properly
  • Hardcoding file paths instead of using path.join()
  • Not validating file paths from user input
  • Forgetting to set encoding when reading text files
