Real-life example: Like a parking lot with numbered spaces. Space 0, Space 1, Space 2... Each space can hold exactly one car.

‚Ä¢ Insert at end: O(1) - Just add it

üîÑ Recursion

What is Recursion?

Think of it like: A function that calls itself to solve smaller versions of the same problem. Like Russian nesting dolls!

Real-life example: Looking for your keys by checking one room, then asking someone else to check the remaining rooms the same way.

Factorial Example: 5!

factorial(5) = 5 √ó factorial(4)
factorial(4) = 4 √ó factorial(3)
factorial(3) = 3 √ó factorial(2)
factorial(2) = 2 √ó factorial(1)
factorial(1) = 1 ‚Üê Base case
Result: 120
// Factorial using recursion function factorial(n) { // Base case - stops the recursion if (n <= 1) { return 1; } // Recursive case - function calls itself return n * factorial(n - 1); } console.log(factorial(5)); // 120 // Fibonacci sequence function fibonacci(n) { if (n <=1) return n; return fibonacci(n - 1) + fibonacci(n - 2); } console.log(fibonacci(6)); // 8
Key Concepts:
‚Ä¢ Base Case: Condition that stops recursion
‚Ä¢ Recursive Case: Function calls itself
‚Ä¢ Stack Space: Each call uses memory
‚Ä¢ Can be slower than iterative solutions

// Bubble Sort - Simple but slow function bubbleSort(arr) { for (let i = 0; i < arr.length; i++) { for (let j=0; j < arr.length - 1; j++) { // Compare adjacent elements if (arr[j]> arr[j + 1]) { // Swap if they're in wrong order [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]; } } } return arr; } // Quick Sort - Much faster function quickSort(arr) { if (arr.length <= 1) return arr; const pivot=arr[arr.length - 1]; const left=arr.filter(x=> x < pivot); const right=arr.filter(x=> x > pivot); return [...quickSort(left), pivot, ...quickSort(right)]; }
Algorithm Time Complexity Best For Description
Bubble Sort O(n¬≤) Learning Repeatedly swaps adjacent elements
Quick Sort O(n log n) General use Divide and conquer approach
Merge Sort O(n log n) Stable sorting Divides array, then merges sorted parts

Think of it like: Finding a specific book in a library, or looking for a friend's phone number in your contacts.

Two main types: Linear (check each item) and Binary (smart jumping for sorted lists).

// Linear Search - Check each element one by one function linearSearch(arr, target) { for (let i = 0; i < arr.length; i++) { if (arr[i]===target) { return i; // Found at index i } } return -1; // Not found } // Binary Search - Only works on sorted arrays function binarySearch(arr, target) { let left=0; let right=arr.length - 1; while (left <=right) { let mid=Math.floor((left + right) / 2); if (arr[mid]===target) { return mid; // Found! } else if (arr[mid] < target) { left=mid + 1; // Search right half } else { right=mid - 1; // Search left half } } return -1; // Not found }
Time Complexity:
‚Ä¢ Linear Search: O(n) - Must check each element
‚Ä¢ Binary Search: O(log n) - Smart jumping, only works on sorted arrays
